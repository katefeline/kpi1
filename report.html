Тема та мета власного WEB-застосунку

Розробка веб-сайту "EcoShop" — інтернет-магазину екологічно чистих товарів.

Мета: Створення семантично розміченого, валідного HTML-документу для представлення e-commerce платформи, що популяризує еко-свідомість та дозволяє користувачам ознайомитись з асортиментом товарів для сталого розвитку.

Опис бізнес-логіки (Предметне середовище)

WEB-застосунок "EcoShop" є вітриною інтернет-магазину. Користувач потрапляє на головну сторінку, де бачить навігаційне меню, промо-банер та інформацію про бренд. На сторінці представлено каталог товарів з цінами та коротким описом. Реалізована функція порівняння характеристик товарів у вигляді таблиці. Користувач може дізнатися про команду, переглянути відповіді на часті запитання (FAQ) та зв'язатися з адміністрацією через спливаючу форму зворотного зв'язку.

Функціональні та нефункціональні вимоги

Функціональні вимоги:

  • Відображення логотипу та назви магазину.
  • Наявність навігаційного меню з посиланнями на розділи (Home, About, Products, Team, Contact).
  • Пошук товарів (поле вводу).
  • Демонстрація списку товарів з картками (назва, ціна, опис).
  • Табличне порівняння характеристик продуктів.
  • Форма зворотного зв'язку у модальному вікні.

Нефункціональні вимоги:

  • Використання семантичних тегів HTML5 (header, main, section, footer).
  • Валідність коду (відповідність стандарту W3C).
  • Доступність (використання атрибутів alt для зображень, aria-label).
  • SEO-оптимізація (мета-теги description, title).

Структура головної сторінки WEB-застосунка

Головна сторінка побудована за допомогою семантичної розмітки HTML5.

    Основні блоки:
  1. <!DOCTYPE html> — оголошення типу документу.
  2. <html> — кореневий елемент з атрибутом мови lang="en"
  3. <head> — метадані, заголовок сторінки, підключення стилів.
  4. <body> — видима частина сторінки, що містить:
    • <header> — "шапка" сайту (лого, пошук, меню <nav>).
    • <main> — основний контент, розділений на секції:
      • <section id="home"> — Hero-банер.
      • <section id="about"> — Інформація про компанію (<article>).
      • <section id="products"> — Картки товарів та таблиця порівняння.
      • <section id="team"> — Інформація про команду (<dl>) та FAQ (<details>).
    • <footer> — "підвал" сайту (контакти, посилання, копірайт).

Робота з таблицями

Теги та атрибути для таблиць

    Для створення таблиць в HTML використовується тег
  • <thead>: Групує заголовки таблиці.
  • <tbody>: Групує основний вміст (тіло) таблиці.
  • <tfoot>: Групує підвал таблиці (підсумки, примітки).
  • <tr>: Визначає рядок таблиці.
  • <th>: Визначає заголовну клітинку таблиці.
  • <td>: Визначає звичайну клітинку таблиці.

Робота із зображеннями

Теги та атрибути зображень

Зображення додаються за допомогою одинарного тегу <img>.

  • <src> (source): Шлях до файлу зображення (обов'язковий). Може бути відносним (./images/logo.png) або абсолютним (URL).
  • <alt> (alternative text): Альтернативний текст. Відображається, якщо картинка не завантажилась, і читається скрінрідерами для людей з вадами зору (важливо для SEO та доступності).
  • <width> та <height>: Задають ширину та висоту зображення в пікселях (рекомендується для запобігання "стрибків" верстки при завантаженні).
  • <loading="lazy">: Дозволяє відкладене завантаження зображень (оптимізація швидкості).

Робота зі списками

Теги та атрибути списків

    В HTML існує три основні види списків:
  1. Маркований список (<ul> - Unordered List): Елементи не мають порядку, позначаються маркерами (крапками). Елементи списку — <li>.
  2. Нумерований список (<ol> - Ordered List): Елементи мають порядок (1, 2, 3 або A, B, C). Атрибути: type (вид нумерації), start (початкове значення), reversed (зворотний порядок). Елементи — <li>.
  3. Список означень (<dl> - Description List): Використовується для термінів та їх описів. <dt> (Description Term): Термін або ім'я. <dd> (Description Details): Опис терміну.
